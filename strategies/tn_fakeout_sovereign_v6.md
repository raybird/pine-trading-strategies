# TN Fakeout Sovereign V6

## 前因後果 (設計動機)
「假突破」是量化交易中最大的磨損來源。本策略的核心目標是建立一套具備「因果一致性校驗」的過濾器，透過 UDT 與狀態機管理，實作對突破信號的高維度物理攔截。

## 技術邏輯 (指標與原理)
*   **對象化審計**：實裝 `AuditContext` 對象，集中管理成交量比、趨勢對位與 Bar 確認邏輯。
*   **生命週期管理**：使用 Enum 規訓將交易流程硬化為 Staging -> Auditing -> Pushing 的 Git 模式。
*   **原生日誌**：全面採用 v6 `log` 建立透明的因果追蹤軌跡。

## 交易規則
*   **進場條件**：信號等級達到 `SOVEREIGN`，且具備 HTF（高位時間框架）的物理背書。
*   **停損獲利**：嚴格的 1:2 風報比，搭配 ATR 波動過濾。
