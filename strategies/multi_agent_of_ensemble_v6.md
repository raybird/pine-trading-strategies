# TN Multi-Agent OF Ensemble Sovereign v6 (多代理人集成訂單流版)

## 策略概述
此策略實體化了 TeleNexus 的「集體智慧」規訓。它不依賴單一的買賣邏輯，而是將訂單流的物理數據分發給三個具備不同專業領域的虛擬代理人進行審計。透過加權集成 (Ensemble) 演算，系統產出一個具備高因果密度的共識得分，作為最終執行的地板，顯著降低了單一模型失效導致的隨機風險。

## 核心特性
- **三位一體代理人系統 (Triple Agent Audit)**:
  1. **Delta Agent (物理淨壓)**: 負責計算成交量增量與總量的比率，評估資金的淨流向強度。
  2. **Imbalance Agent (矩陣失衡)**: 遍歷 v6 `footprint` 物理行，偵測買賣力量在價格層級上的顯著不對稱。
  3. **POC Agent (價值定位)**: 監控控制點 (POC) 的動態位移，判定當前價格相對於價值核心的位置。
- **加權集成演算 (Ensemble Scoring)**: 透過自定義權重（$W_{delta}, W_{imb}, W_{poc}$）合成統一的主權得分。這允許使用者根據市場屬性動態調整不同代理人的話語權。
- **三級感知規訓 (Sensing Regimes)**:
  1. **Sovereign (0.5)**: 需三個代理人達成強共識，確保最高的執行品質。
  2. **Standard (0.3)**: 標準規訓模式，適合平衡型交易。
  3. **Pulse (0.1)**: 高頻感知模式，捕捉微小的共識偏移。
- **高時框規訓對位 (HTF Alignment)**: 集成信號必須與日線級別的大趨勢方向物理一致，杜絕逆勢操作。

## 模組說明
1. **Perception Hub**: 負責獲取並分發原生 v6 足跡數據。
2. **Agent Specialists**: 包含三個獨立的 `method` 演算法。
3. **Decision Resolver**: 執行加權求和與規訓門檻判定。
4. **Ensemble HUD**: 實時披露各代理人的分值與最終的集成結果。

## 使用建議
- **主週期**: 建議在 15M、30M 或 1H 週期使用。
- **權重配置**: 若市場震盪劇烈，建議增加 **POC Agent** 的權重；若趨勢明確，則增加 **Delta Agent** 權重。
